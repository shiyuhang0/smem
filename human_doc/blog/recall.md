# Recall

长期记忆系统 recall。

## 方案

假设需要召回n(5)条记忆，设计如下流程：

粗排：
1. 向量搜索4n（20） + 全文检索4n（20）
2. 头部取值+RRF 融合：各取头部前n(5)名。剩下各3n(15)个 RRF融合，k = 2n (10-60)，得 top4n（20）。一共得到 4n～5n 条候选。
精排：
1. bge-rerank-v2-m3 rerank + 阈值过滤（0.6）: 实际测试基本1s 内即可 rerank 完。
2. boost 分数：相似性为主，但会 boost 其他维度，如时间（近期优先）、存储次数（多次记忆优先）、类型（和问题匹配的类型优先）等。权重比相似性低不少。
3. 可选：top 2n + softmax + temperature：对剩余结果的相关性得分进行 softmax 归一化，得到每条结果被召回的概率。引入 temperature （纯玩，可以发散记忆，不至于每次选到同样的结果，给其他记忆一些机会）
4. 选分高的 topn。

## 混合检索

混合搜索即向量搜索+全文检索
- 向量搜索(vector search)：Embedding 后基于向量相似度进行检索。选择余弦相似度，方便评估相似度计算。擅长语义理解，如番茄可以召回西红柿。
- 全文检索(FTS)：基于倒排索引查询 + BM25 算法排名。擅长精确匹配文本。

混合检索的好处：兼顾 语义理解 与 精确匹配。
- 向量搜索的优势（语义）： 能够理解意图。搜索“不开心”，它可以找回“心情低落”，即便关键词不重合。
- 全文检索的优势（字面）： 能够精准锁定。查"ID 为 9527 的订单"，全文检索能通过 BM25 等算法实现极高的准确性。因为向量模型很难精准区分 9527 和 9528。
- 数据量越大（万），混合检索优势约明显。

为什么头部取值+RRF 融合：

RRF 的特点是长尾衰减，排名越靠后权重越低。1 + 无排名 < 10 + 10。因此：

- 优点：适合共识，排除单路噪声，去除 “伪正例”
  - 希望去除全文检索的“伪正例”： 用户搜“苹果笔记本”，结果排第一的是一篇讲“如何用笔记本记录如何吃苹果”的流水账。关键词全中，但语义不对。
  - 希望去除向量搜索的“伪正例”： 向量模型可能觉得“猫”和“狗”的语义很近。你搜“猫的护理”，它可能把“狗的训练”排在第一。语义接近，但关键实体错了。
- 缺点：无法互补，单路第一可能被两路最后挤掉。
  - 如向量搜索“番茄”，它可以找回“西红柿”。但全文检索搜不到 “西红柿”。西红柿一下子会被 RRF 排除掉！

因此 RRF 可以排除噪声，但也可能错失单路最佳匹配。由于我后面还有 rerank，我可以让噪音进来，但不希望放过单路第一。因此先头部取值，然后再用 RRF 融合剩余的结果，兼顾单路保护和共识融合。

## 为什么 rerank 得分

> 没有其他纯数学的好办法。

正常方案可能是 0.7 vectore score + 0.3 bm25 score。

- 基于余弦相似度的向量搜索，结果是有意义，越接近1越相关。
- 直接 bm25 不可行：是一个分数，不是 [0,1]，量纲不同。
- bm 25 归一化不一定可行：
  1. 得分无 **最大值**，因此得分只能看出相对相关性。比如 20分，1分。那么知道20分更相关。但不知道有多相关。
  2. 归一化失去意义：归一化是为了统一量纲，赋权计算得分。向量搜索在 0-1，1 表示最相关。但 BM25 归一化到 0 到 1 后，1 只表示相对最相关，实际可能相关性很低。举个例子，向量搜索 0.01, 0.01，BM25 归一化就可能得到 0.5，0.5，最后俩得分都不低，实际毫无关系。

before: 
- vector search + 全文检索 + RRF 融合（k=60） 。打分： 0.7 vector + 0.3 bm25。
- topk = 2 的情况下。 RRF 融合直接把爱吃番茄挤掉了，因为全文搜索里根本没有他,但 vector search 里其实他排第1。
- 最终输出：看了西红柿相关小说 + 看了西红柿首富电影

now
- 当前方案
- topk = 2 的情况下。 RRF 融合前取了头部，爱吃番茄未被挤掉
- rerank 打分后，爱吃番茄直接高分 0.86，其他的均不到 0.6，直接被过滤掉了。

## Lesson Learned

1. 不依赖 RRF，无法发挥两种搜索互补的能力，一路搜出来，另一个搜不出来的情况会被直接排除掉。
2. 打分不能 vector 和 bm25 加权。bm25 不是 [0,1] 的分数，归一化后也不一定有意义。 rerank 模型很必要。
3. 业务上 boost，帮助相关性相近的记忆区分出业务相关的优先。比如记忆中近期的和多次被记忆的更相关一些。

## Ref

向量搜索
- 余弦相似度：衡量两个向量的夹角，适合文本相似度。
- 欧氏距离：衡量两个向量的距离，适合数值数据。
- 内积：衡量两个向量的乘积，适合高维稀疏数据。
- 曼哈顿距离：衡量两个向量在各维度上的绝对距离之和，适合稀疏数据。

全文检索倒排索引
1. 分词：将文本分解成单词或短语。如定义全文索引就指定分词器 `FULLTEXT INDEX (title) WITH PARSER MULTILINGUAL`
2. 标准化：统一大小写、去除停用词等。
3. 统计位置: 记录每个词在文档中的位置和频率，用于后续的“短语匹配”和“相关性打分”。
4. 倒排列表写入：将每个词和它出现的文档列表写入索引文件，支持快速查询。

全文检索查询过程
1. 查询分析：对搜索词进行同样的分词和标准化，得到多个关键词
2. 词条检索：直接在倒排索引的词典里找到对应的关键词
3. 查询文档：根据倒排列表获取包含关键词的文档集合
4. 排序 (Ranking): 根据相关性打分算法（如 TF-IDF、BM25）计算各文档的相关性分值,并返回排序后的结果列表。

全文检索 BM25
- TF (Term Frequency, 词频)：随着词频增加，得分会迅速上升，但随后会趋于一个极限（饱和点）
- IDF (Inverse Document Frequency, 逆文档频率)：关键词越稀有（比如“量子”比“的”稀有得多），权重越高。
- 文档长度归一化：如果一个词在短文档中出现 3 次，其权重比在万字长文中出现 3 次更高。

BM25 公式 

$$Score(D, Q) = \sum_{i=1}^{n} \text{IDF}(q_i) \cdot \frac{f(q_i, D) \cdot (k_1 + 1)}{f(q_i, D) + k_1 \cdot (1 - b + b \cdot \frac{|D|}{avgdl})}$$

- 最小得分：0
- 最大得分：不确定。得分是一个相对值，取决于你的查询词个数、IDF 的值以及 $k_1$ 等参数的设定。

RRF 公式。

$$RRFscore(d) = \sum_{r \in R} \frac{1}{k + rank(r, d)}$$

- $R$：所有的结果列表集合（例如一个 BM25 列表，一个向量检索列表）。
- $rank(r, d)$：文档 $d$ 在列表 $r$ 中的排名（第 1 名、第 2 名...）。
- $k$：一个调节常数。默认值通常取 60。
- 最小值0，最大值受 k 影响。

**RRF 的真实意义** ：
  
具备长尾衰减：权重随着排名（Rank）下降而迅速降低，且下降速度越来越慢。每一路的头部分高，后面断崖下跌。

因此其作用在于去除某路的噪声，共识融合。比如在单路第一另一路不存在，得分不如在两路都存在但排名都第10的。最终结果就是会得到**共识**，而非单路极端。

**RRF k值**

- $k$ 越小，头部越强： 当你对每个检索器的“头部结果”非常有信心时
- $k$ 越大，更看重“共识”： 排名第 1 和第 10 的权重差异缩小，希望通过多个检索器的交集来过滤噪音，追求稳定性时
- k = 60: 兼顾头部和共识，适合大多数情况。排名 60 为第一的一半。
- k 和 每路数量 x 的关系：x 小，则 k 不宜太大。否则第一和第 x 差异太小。

| 检索场景 | 每路召回深度 (Top-N) | 推荐 $k$ 值 | 融合截断 (Top-K) | 配置逻辑与效果 |
| :--- | :--- | :--- | :--- | :--- |
| **极小样本/垂直精准检索** | 10 - 20 | **1 - 10** | 10 | **头部强保护**：确保单路第一名的权威，不被后排的“平庸交集项”反超。 |
| **中等规模混合搜索 (SaaS/APP)** | 50 - 100 | **30 - 60** | 50 | **平衡型**：兼顾单路准确率与多路共识，是大多数搜索业务的基准配置。 |
| **RAG 知识库/语义检索** | 100 - 200 | **60** | 100 | **标准平滑**：利用 $k=60$ 的长尾特性过滤向量检索中的语义漂移噪音。 |
| **高召回/多路粗排阶段** | 500 - 1000 | **100 - 120** | 200 | **稳定性优先**：拉平头部差距，筛选在多路检索中表现均稳定的长尾候选集。 |
| **多路完全互补场景** | 50 - 100 | **100+** | 50 | **共识优先**：极大程度压制单路第一，只有在两路中均被搜到的结果才会排在最前。 |

BGE-Reranker

- 是什么：二分类模型，输入是 query 和 candidate 的文本，输出是相关置信度。
- 输出：可进行深层语义匹配，输出的是范围通常在 $[-10, 10]$。一般通过 Sigmoid 归一化后设置阈值(0.6～0.75比较可信)。
- 最佳实践：输入 30～60 条。bge-rerank-v2-m3 50条 mac 1-2 秒。
